TP 2 : Maîtrise de cURL

Testez les commandes cURL suivantes dans votre terminal.
curl -i https://httpbin.org/get
curl -v https://httpbin.org/get
Question : Quelle est la différence entre -i et -v ?
• -i affiche les headers de la réponse HTTP avant le corps.
• -v (verbose) affiche tout le processus : connexion, SSL, headers envoyés et reçus.

TP 3 : API REST & JavaScript

Implémentation d'une fonction de résilience réseau (Retry).
async function fetchWithRetry(url, options, maxRetries) {
    for (let i = 0; i <= maxRetries; i++) {
        try {
            const response = await fetch(url, options);
            if (response.status >= 500) throw new Error(`Erreur ${response.status}`);
            return response;
        } catch (err) {
            if (i === maxRetries) throw err;
            await new Promise(res => setTimeout(res, 1000));
        }
    }
}

Exercice 2 : Questions théoriques

1. Différence entre no-cache et no-store ?
no-cache oblige à valider la ressource auprès du serveur. no-store interdit tout stockage.

2. Pourquoi POST n'est-il pas idempotent ?
Car plusieurs requêtes identiques créent plusieurs ressources distinctes (ex: deux commandes au lieu d'une).

3. Que se passe-t-il si le serveur renvoie un code 301 ?
Le navigateur effectue une redirection permanente vers la nouvelle URL spécifiée.

4. À quoi sert le header Origin ?
Utilisé pour le CORS, il indique la source (domaine) de la requête.

5. Pourquoi utiliser HttpOnly sur les cookies ?
Pour empêcher le vol de session via JavaScript (attaques XSS).
