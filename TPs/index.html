TP 2 : Maîtrise de cURL

Utilisez les options -i et -v pour observer les échanges.
Question : Quelle est la différence entre -i et -v ?
L'option -i affiche uniquement les en-têtes (headers) de la réponse. L'option -v (verbose) affiche tout le cycle : connexion, en-têtes envoyés et en-têtes reçus.

Exercice 2 : Questions théoriques

1. Quelle est la différence entre no-cache et no-store ?
no-cache signifie que le navigateur doit valider la ressource avec le serveur avant de l'utiliser. no-store signifie que rien ne doit être mis en cache du tout.

2. Pourquoi POST n'est-il pas idempotent ?
Car l'envoi répété d'une requête POST peut créer plusieurs ressources ou avoir des effets secondaires multiples (ex: deux paiements).

3. Que se passe-t-il si le serveur renvoie un code 301 ?
Le navigateur redirige automatiquement l'utilisateur vers la nouvelle adresse permanente.

4. À quoi sert le header Origin ?
Il indique au serveur d'où provient la requête (CORS) sans envoyer d'informations sensibles.

5. Pourquoi utiliser HttpOnly sur les cookies de session ?
Pour empêcher que les scripts JavaScript malveillants (XSS) ne puissent lire le cookie de session.
