TP 2 : Maîtrise de cURL

Différence entre -i et -v :

L'option -i affiche uniquement les en-têtes de la réponse. L'option -v (verbose) affiche tout le tunnel de communication (headers envoyés ET reçus, IP, handshake SSL).

Exercice Avancé :

curl -i -X POST \
  -H "Content-Type: application/json" \
  -H "X-Custom-Header: MonHeader" \
  -d '{"action": "test", "value": 42}' \
  https://httpbin.org/post

TP 3 : API REST avec JavaScript

Script fetchWithRetry :

async function fetchWithRetry(url, options, maxRetries) {
  for (let i = 0; i <= maxRetries; i++) {
    try {
      const response = await fetch(url, options);
      if (response.status >= 500) throw new Error("Erreur Serveur");
      return await response.json();
    } catch (err) {
      if (i === maxRetries) throw err;
      console.log(`Tentative ${i+1} échouée. Retry...`);
      await new Promise(resolve => setTimeout(resolve, 1000));
    }
  }
}

Exercices Récapitulatifs - Questions Théoriques

  1. Différence no-cache vs no-store : no-store interdit tout stockage. no-cache autorise le stockage mais force la validation auprès du serveur avant usage.
  2. Idempotence du POST : POST n'est pas idempotent car répéter la requête crée généralement plusieurs ressources (ex: deux commandes validées).
  3. Code 301 : Indique une redirection permanente. Le navigateur mémorise la nouvelle URL.
  4. Header Origin : Indique au serveur d'où provient la requête pour gérer les politiques CORS.
  5. HttpOnly : Empêche JavaScript d'accéder aux cookies, protégeant ainsi contre le vol de session via XSS.
